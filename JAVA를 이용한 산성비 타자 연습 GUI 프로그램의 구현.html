1.   서   론

 

한컴타자연습에 나오는 산성비 타자 연습 프로그램을 만든다. 기존에 한글 단어만 나오던 산성비 타자연습에 영문 타자를 연습할 수 있도록 영어 단어도 추가한다.

프로그램 설계 단계에서 작성한 것을 바탕으로 프로그램을 구현하여 동작해보고 결과를 확인한다.

3.   프로그램 설계

 

초기 화면을 설정한다. 초기 화면에서 언어설정과 난이도 설정을 가능하게 한다. 언어는 한글과 영어 두가지로 한다. 난이도 설정은 한글과 영어 각 1단계 부터 10단계로 한다. 초기화면에서 언어와 난이도 설정 후 ‘START’버튼을 누르면 언어와 난이도에 맞는 게임화면으로 전환된다. 초기화면에서 게임종료 버튼을 누를시 프로그램이 종료된다.

 

게임 화면 에서는 초기화면에서 설정한 언어와 난이도에 따라 산성비 게임에서 내려올 단어의 종류와 단어의 속도, 단어의 생성속도를 조절한다. 게임화면의 왼쪽 상단에는 설정한 난이도, 중앙 상단에는 현재 점수, 우측 상단에는 남은 목숨, 그 바로 아래에 시도 횟수, 중앙 하단에 입력창, 우측 하단에는 초기화면과 게임종료 버튼을 생성한다. 단어는 단어장에 있는 단어들을 무작위로 배치해 각 난이도에 맞는 속도로 생성되어 내려오고 내려오는 단어들이 바닥에 닿으면 목숨이 하나 소모된다. 총 입력은 100번 할 수 있고, 정답과 상관없이 100번을 입력할 수 있기에 오답을 입력할 경우 최고 점수가 낮아진다. 100번을 다 입력한 경우와 게임종료 버튼을 입력한경우, 남은 목숨이 0이 된 경우에는 결과화면으로 전환된다.

 

   결과 화면 에서는 목숨이 남아있고 입력횟수가 100이 되었거나, 게임화면에서 게임종료 버튼을 눌렀을 경우 화면에 “WIN”과 함께 점수가 출력되고 가장 아래에 초기화면 버튼이 생성된다. 반면, 목숨이 모두 소모되어 결과화면으로 넘어오게 된다면 화면에 “Game Over”와 함께 점수가 출력되고 가장 아래에 초기화면 버튼이 생성된다. 게임이 끝나고 결과 화면에서 초기화면 버튼을 통해 초기화면으로 돌아가 언어와 난이도를 재설정 후 게임을 다시 실행할 수 있고, 게임종료버튼을 클릭해 프로그램을 종료할 수 있다.

 

 

 

 

 

4.   프로그램 구현

 

       i.          AcidRain.java

초기화면(메인화면)인 AcidRain_Screen()을 실행한다.

 

 

     ii.          AcidRain_Screen.java

메인화면을 구현해둔 파일로,

GridLayout을 통해 2x3 배치를 해주고, 각 그리드 사이의 간격은 15로 고정해준다.

JLabel을 통해 1행 좌측에는 “언어” 를 표시해주고 JComboBox를 통해 오른쪽 박스에서 원하는 언어(한글, 영어)를 선택할 수 있도록 한다.

JLabel을 통해 2행 좌측에는 “난이도” 를 표시해주고 JComboBox를 통해 오른쪽 박스에서 원하는 난이도(LEVEL 1 ~ LEVEL 10)를 선택할 수 있도록 한다.

창의 맨 밑 행에는 JButton으로 START 버튼과 게임종료 버튼을 생성해준다.

START 버튼 클릭시 ActionListener를 통해 선택한 언어와 난이도에 해당하는 값을 받아 게임화면(AcidRain_GameScreen(난이도, 언어))을 실행시키고 현재화면은 setVisible(false)로 안보이게 한다.

게임종료 버튼 클릭시 System.exit(0);을 실행시켜 프로그램을 종료한다.

 

 

   iii.          AcidRain_GameScreen.java

메인화면에서 선택된 언어와 난이도를 받아 실행된다.

전체 화면은 1200*900픽셀로 해준다. 화면의 좌측상단에는 JLabel을 사용해 현재 선택한 난이도를 표시한다. 선택한 언어가 한글이면 “0 단계”로 표시되고, 영어라면 “Level 0”으로 표시된다.

화면의 중앙 상단에는 JLabel을 사용해 현재 점수를 표시하고 우측 상단에는 JPanel을 통해 빨간색 사각형을 3개 생성해 남은 목숨을 표시해준다. 남은 목숨 바로 아래에는 정답시도횟수를 표시해준다. 화면의 하단에는 JPanel을 사용해 게임중 단어가 내려와 닿으면 사라지는 기준점(y=590)이 될 바닥을 생성해준다. 화면의 중앙 최하단에는 JTextField를 사용해 정답을 입력할 수 있는 텍스트필드를 생성한다. 텍스트필드에서 설정한 ActionListener는 필드값을 받아 removeAnswer();을 실행하고 필드를 “”로 초기화 한다. 화면의 우측 하단에는 JButton을 사용해 메인화면으로 돌아가는 버튼(초기화면)과 게임을 종료하는 게임종료버튼을 생성한다.

Word.java에 포함되어있는 word_create.create(언어);를 통해 언어에 맞는 단어리스트를 생성하고 word_create.shuffle();을 통해 매번 다른 문제 출제를 위해 랜덤으로 셔플해준다.

new AcidRain_Thread().start();를 통해 스레드를 실행시킨다.

AcidRain_Thread class는 각 난이도에 맞게 속도를 설정해 단어들을 무작위로 x축에 배치한다. 이때 단어들은 계속 비처럼 내려오기만 해야하므로 y축은 생성되는 단어들이 계속 같은 y값을 가져야한다. 단어가 label 패널에 추가되면 AcidRain_Move().start();를 통해 단어들의 y값을 변화시킨다.

AcidRain_Move class에서는 각 label[i]의 x, y의 좌표값을 저장하고, y좌표값을 증가시킨 후 label[i].setLocation(x,y) 를 통해 위치를 다시 지정해준다. 이때 다시 지정한 y의 좌표값이 아까 설정한 바닥(y좌표가 590)보다 크다면 life 를 1 감소시키고 해당 label의 상태를 안보이게 설정한다. 이때 life의 값에 따라 줄어든 life의 수만큼 빨간색 목숨을 회색으로 바꿔준다. 이때 남은 목숨이 0이라면, 현재화면(게임화면)을 setVisivle(false)를 통해 가리고, new AcidRain_EndScreen(점수, 목숨); 을 통해 실행한다.

removeAnswer() 함수는 입력된 값이 리스트 값과 일치한다면 해당 레이블을 안보이게 하고, 점수를 5점 증가시킨 후 setText();를 통해 화면에 나오고 있는 현재점수를 갱신시킨다. 일치여부와 관계 없이 removeAnswer() 함수가 동작될 때마다 removeCount(정답시도횟수)를 1 증가시킨다. 만약 removeCount가 100이 된다면, 현재화면(게임화면)을 가리고 결과화면을 new AcidRain_EndScreen(점수, 목숨); 을 통해 실행한다.

 

 

   iv.          AcidRain_EndScreen.java

게임화면에서 현재 목숨과 점수를 받아 실행된다.

만약 남은 목숨이 0이라면 화면에 “Game Over”와 “game over… x 점”(현재점수), 초기화면 버튼이 생성된다.

남은 목숨이 1 이상이거나, 게임종료 버튼을 통해 해당 화면으로 넘어오게된 경우, 화면에 “WIN”과 “ x 점”(현재점수), 초기화면 버튼이 생성된다.

 초기화면 버튼 클릭시 AcitonEvent를 통해 해당 엔딩화면은 setVisivle(false)를 통해 가려지고, new AcidRain_Screen();을 실행시킨다.

 

 

     v.          Word.Java

create(언어) 함수는 게임화면에 전달된 언어값을 받아와 실행된다. 언어 타입이 한글이라면 Kor.txt 파일에서 각 줄마다 적힌 단어를 읽어 arr에 저장하고, 영어라면 Eng.txt 파일에서 단어를 읽어와 arr에 저장해준다.

 shuffle() 함수는 저장된 arr의 값들이 매번 같은 순서를 같지 않도록 하기위해 Collections를 사용해서 무작위로 섞어준다.

Kor.txt : 한글 단어들을 저장하고 있는 txt파일

Eng.txt : 영어 단어들을 저장하고 있는 txt파일

6.   결과 분석 및 토의

   5장의 동작 결과가 이상없이 정상적으로 수행된 것인지 확인한다.

n  오답을 입력하였을 경우 점수가 증가하는지 확인한다.

오답을 입력하였을 경우 점수가 증가하지 않고, 시도횟수는 증가했으므로 정상적으로 작동함을 알 수있다.

 

 

n  정답을 입력하였을 경우 label이 정상적으로 사라지는지 확인한다.

 

정답을 입력하였을 경우 label이 정상적으로 사라지므로 정상적으로 작동함을 알 수있다.

하지만, 정답을 입력하여 label이 사라졌다고 하더라도 Jlabel의 상태가 setVisible(false)상태일 뿐, 해당 label이 사라진 것이 아니기 때문에 다시 입력하면 정답으로 인식되어 점수가 오르는 현상이 발생한다는 것이다.

또, 간헐적으로 정답을 입력하였을 경우 정답처리는 이루어지지만, 시도횟수와 텍스트필드가 초기화되지않는 경우도 발생했다. 5 - iii를 보면 시도횟수가 100회이고 정답을 맞췄을 경우 5점이 추가되기에 최고점수는 500점인데 510점이 나온것을 통해 알 수 있다.

 

n  시도횟수가 100을 채웠을 경우 정상적으로 결과화면이 출력되는지 확인한다.

5 – iii 에서 볼 수 있듯, 결과화면에 “WIN” 과 “x 점”, 초기화면 버튼이 출력되므로 정상적으로 작동함을 알 수있다.

7.   결론

 

 프로그램을 만드는데 있어서 정상적인 작동까지 많은 시간이 소요되었다. 6.에서 언급한 오류들을 수정하려고 하였지만 역량이 받쳐주지 못하여 역부족이였다. 소나기 프로그램 작성을 위해 블로그나 git에 있는 많은 소스코드를 읽어보면서 아직 많이 부족하다는 사실을 깨달았다. 향후 java에 대해 더 많이 공부하고 시간이 된다면 이번에 만든 산성비 프로그램을 보완해보고싶다. 또, 다른 소스코드들은 이번 프로그램처럼 각 화면에 대해 파일을 나누지않고 GUI, Play, Rian, … 등으로 구성해서 나눈게 대부분이였다. 이런 이유는 잘 모르겠지만 차차 공부하고 발전하다보면 이러한 이유를 깨닫게되어 자연스럽게 프로그램을 구성하는 날이 찾아오지않을까 기대한다.
